# Roulette
Рулетка событий на трансляцию для работы с Streamer.bot

# Настройка Streamer.bot

    Создай новое действие (Action).

    В триггерах выбери Channel Reward Redemption.

    Выбери конкретную награду (например, "Рулетка баффов").

    Добавь действие типа Send HTTP Request.

        Метод: POST

        URL: http://localhost:8080/api/spin

        Body (JSON):
        json

        {
          "roulette": "buffs"
        }

        (для других наград используй другие имена рулеток, например debuffs).

    При желании можно передавать имя пользователя, но в нашем случае не обязательно.

    Убедись, что сервер запущен и доступен из сети (если стримерский ПК и сервер на том же компьютере, используй localhost или 127.0.0.1). Если на разных машинах – укажи IP.

# Запуск и использование

    Установи Go, склонируй проект.

    Создай папку configs и положи туда JSON-файлы с событиями.

    Запусти сервер: go run .

    Открой браузер по адресу http://localhost:8080 – это и будет оверлей. Его нужно добавить как источник браузера в OBS (с прозрачным фоном).

    Когда в чате активируется награда, Streamer.bot отправит запрос, сервер выберет случайное событие и передаст его на оверлей. Тот анимирует вращение и покажет результат.

# Запуск в docker-контейнере

    Скопируй файлы в корень проекта.

    Собери и запусти:

bash

> docker-compose up -d --build

    Проверь, что сервер работает:

        Оверлей доступен по адресу http://localhost:8080 (в браузере или как источник в OBS).

        Эндпоинт для Streamer.bot: http://localhost:8080/api/spin (POST).

    Изменение конфигов/статики:

        Правишь файлы в папках configs/ или static/ на хост-машине.

        Контейнер автоматически подхватит изменения (благодаря volume).

        Перезагружать контейнер не требуется (кроме случаев, когда меняется логика Go-кода – тогда пересборка).

    Остановка:

bash

> docker-compose down
